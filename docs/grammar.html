Rédiger des tests fonctionnels en langage naturel

Ce projet permet de décrire des scénarios de tests dans un français simple puis de générer un script shell correspondant. Chaque ligne d'un fichier de test suit la forme :

Action: <description de l'étape> ; Resultat: <résultat attendu>

Verbes et éléments reconnus

  • Initialisation : initialiser, créer, configurer
  • Exécution : exécuter, lancer, traiter
  • Validation : vérifier, valider
  • Fichiers : toucher, cat, créer fichier/dossier

Des paramètres optionnels peuvent être ajoutés sur les lignes :

  • argument NOM=VALEUR – ajoute un argument au batch
  • chemin des logs = /chemin/fichier.log
  • script sql = fichier.sql
  • créer ou mettre à jour fichier/dossier = CHEMIN avec les droits = MODE
  • toucher le fichier CHEMIN [-t TIMESTAMP]

Exemple complet

Action: toucher le fichier /tmp/init.flag 202501010000 ; Resultat: fichier initialisé
Action: exécuter /opt/batch/traitement.sh avec argument produit=42 ; Resultat: traitement réalisé
Action: vérifier qu'il n'y a pas d'erreurs dans les logs ; Resultat: aucun message d'erreur

Utilisation

  1. Créez vos fichiers de test dans le dossier tests/ en suivant la forme ci-dessus.
  2. Lancez python generate_tests.py --batch-path /chemin/vers/batch.sh. Chaque fichier .txt génère un script shell dans le dossier output/.
  3. Exécutez ensuite le script shell généré pour reproduire les étapes.

Un fichier syntaxes/nltest.tmLanguage.json est fourni pour ajouter une coloration syntaxique dans Visual Studio Code.
